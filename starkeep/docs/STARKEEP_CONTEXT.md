## 1. What This Product Is

Starkeep Academy is a **cross-platform gamified education platform** (web + iOS + Android) where real-world heroic actions become an academic record visualized as a Star Map. Students create a heroic alter-ego (Avatar), complete real-world Missions, earn stars that cluster into Constellations, and accumulate LUX currency proportional to their measurable social impact.

**The 10-year vision** includes VR campuses, holographic UIs (the Atlas), live e-sports exams, and a full virtual city (City of Light). **v1 ships none of that.** The manifesto is the compass; the wireframes are the contract.

## 5. The LVM Formula (Implement Exactly As Written)

```
RawScore = (I × 0.30) + (S × 0.25) + (U × 0.20) + (R × 0.15) + (H × 0.10)
BaseLUX  = RawScore × SCALE   (default SCALE = 5)
IssuedLUX = floor(BaseLUX × VSM)

Hard cap: max 30 LUX per milestone
```

### Scoring Axes (0–5 each)

| Axis                     | Weight | Meaning                            |
| ------------------------ | ------ | ---------------------------------- |
| I — Impact Longevity     | 30%    | How long does the benefit persist? |
| S — Scope of Benefit     | 25%    | How many people / how deep?        |
| U — Urgency of Need      | 20%    | How critical is this intervention? |
| R — Rarity & Innovation  | 15%    | How novel is this solution?        |
| H — Human Effort & Skill | 10%    | How much labor and expertise?      |

### Validation Strength Multiplier (VSM)

| Evidence Quality                       | Multiplier |
| -------------------------------------- | ---------- |
| Basic (timestamped photo, video, text) | 1.00       |
| Peer/community confirmation            | 1.10       |
| Verified NGO / accredited witness      | 1.15       |
| Third-party certification              | 1.25       |

### First 5 LUX Auto-consumed for Level-Up

- Every milestone validation: first 5 LUX → consumed for 1 Level (non-cashable)
- User can configure a higher portion to go to leveling
- Level gates: Guild access, voting weight, premium job tiers, visibility

```
  "version": "1.0",
  "quiz_run_id": "qr_abc123",
  "completed_at": "2026-04-25T14:23:00Z",
  "results": {
    "sun_sign": "aries",
    "moon_sign": "cancer",
    "rising_sign": "capricorn",
    "jung_archetype": "magician",
    "mbti": "INFP",
    "recommended_heroic_path": "dreamwalker",
    "recommended_learning_path": "divergent",
    "purpose_seed": "Self-Actualization Architect"
  },
  "raw": { }
}
```

Payload is HMAC-signed. Store raw in `archetypeprofile.raw_quiz_output` (JSONB).
Validated with JSON schema before write. Emits `ArchetypeUpdated` signal.

## 13. The LUX Two-Charge Economy

```
User completes heroic action
  ↓
Admin validates (v1) → LVM formula runs → IssuedLUX computed
  ↓
First 5 LUX → Level-up consumption (non-cashable, removed from supply)
Remainder → wallet.positive_balance (LUX+)
  ↓
v2: User transfers LUX+ to another avatar
  → Sender's LUX+ decreases
  → Recipient gains LUX- (now cashable)
  ↓
v2: LUX- can be converted to fiat (via Stripe)

v1 only implements: earning LUX+, level-up consumption, read-only display
v1 stubs: transfer (501), donate (501), cashout (501)
```
